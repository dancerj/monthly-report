\begin{prework}{ 吉田@小江戸 }

自宅で日常的に活用しているファイルシステムはReiserFSです。
仕事で使ってる環境はext3ですが、ext3は電源断等でジャーナルが
壊れて壊滅した過去の経験（古いカーネルですが...）があり、
あまり信用していません。
自宅のReiserFS環境ではこれまでの所いきなり電源を切ったり、
夏にHDDが壊れかけたりしても被害を被った経験が無いので。
継続的に使っています。
数年前はReiserFSのメイン開発者(Hans Reiser)が逮捕されてしまい、メンテナンスを心配していました。
しかし、その後もReiserFSは他の開発者により継続して保守されているので、安心しました。

\end{prework}

\begin{prework}{ まえだこうへい }
\begin{itemize}
 \item Debianでは特に凝ったことせず、ext3を使ってます。OpenBlockSにしろ、
       MacBookにしろ、利用するデータ量は経験的に分かっているので、LVMは
       使ってません。LVMを使うのはKVM/QEMUを使うときに、qcow2イメージディ
       スクで仮想マシンを作るときだけです。
 \item うちのなかで一番特殊なファイルシステムは、Armadillo-Jで自宅内の
       DHCPサーバを運用するためにJFFSを使っています。これのデフォルトの
       ファームウェアでは、リブートするとすべて初期化されてしまうので、
       RAM領域に書きこんで、電源落としても消えないJFFSが便利です。Debian
       ではないので、直接関係ないですが、DebianのARM版のudhcpのパッケー
       ジを使ってます。
       \footnote{\url{http://d.hatena.ne.jp/mkouhei/20080601/1212330630}}
 \item Debian絡みで、今、自分の中で一番ホットなのはpalm webOSです。これはUbuntuを
       カスタマイズしたものらしいのですが、/etc/mtabを見ると35行もあると
       いうかなり変態な構成になってます。
\end{itemize}
\end{prework}
